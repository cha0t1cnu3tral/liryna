# liryna

A scaffolded game/project repository ready for implementation.

## Project Layout

```
assets/       Game/application assets
build/        Local build outputs (ignored by git)
data/         Data files used by runtime/tools
docs/         Design docs, specs, and notes
include/      Public headers
saves/        Local save data (ignored by git)
src/          Source code
tests/        Test sources
third_party/  Vendored dependencies
tools/        Scripts and utility tools
```

## Getting Started

1. Clone the repo.
2. Create a feature branch from `main`.
3. Add implementation in `src/` and headers in `include/`.
4. Add tests in `tests/`.
5. Open a PR with a short summary and testing notes.

## Conventions

- Keep generated and machine-local files out of git.
- Prefer small, focused commits with clear messages.
- Document decisions in `docs/` as the project evolves.
