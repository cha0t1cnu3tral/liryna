# Contributing

## Workflow

1. Create a branch from `main`.
2. Make focused changes.
3. Add or update tests when behavior changes.
4. Open a PR with:
   - What changed
   - Why it changed
   - How it was tested

## Commit Messages

Use concise, imperative commit messages. Example:

`Add initial save-file parser`

## Quality Bar

- Code compiles without warnings where practical.
- Tests pass locally for changed areas.
- New behavior is documented in `docs/` when needed.
